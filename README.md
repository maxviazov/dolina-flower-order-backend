# Dolina Flower Order Backend

Серверная часть системы заказа цветов для компании Dolина. MVP версия с использованием SQLite для быстрого старта.

## Описание проекта

Система предназначена для:
1. Предоставления клиентам возможности формирования заказов на поставку цветов через онлайн-форму
2. Обработки заказов специалистами компании
3. Формирования Excel-таблиц для отправки заказов на цветочную ферму

## Архитектура

Проект построен с использованием чистой архитектуры:
- **Domain** - бизнес-логика и модели данных
- **Handlers** - HTTP обработчики
- **Services** - бизнес-сервисы
- **Repository** - работа с данными
- **Config** - конфигурация приложения
- **Logger** - структурированное логирование

## Структура проекта

## Быстрый старт

### Предварительные требования

- Go 1.25.3+

### Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/maxviazov/dolina-flower-order-backend.git
cd dolina-flower-order-backend
```

2. Установите зависимости:
```bash
go mod download
```

3. Запустите приложение:
```bash
go run cmd/server/main.go
```

Сервер запустится на `http://localhost:8080`

### Тестирование

См. документацию в директории `docs/` для примеров API запросов и тестирования.

## Конфигурация

Приложение поддерживает конфигурацию через переменные окружения. Основные параметры:
- `SERVER_HOST` - хост сервера (по умолчанию: localhost)
- `SERVER_PORT` - порт сервера (по умолчанию: 8080)
- `LOG_LEVEL` - уровень логирования (info, debug, warn, error)

Полный список см. в `docs/config.example.json`.

## API Endpoints

### Health Check
- `GET /health` - проверка состояния сервиса

### API v1
- `GET /api/v1/ping` - тестовый endpoint
- `GET /api/v1/flowers` - список доступных цветов
- `POST /api/v1/orders` - создать заказ
- `GET /api/v1/orders/:id` - получить заказ по ID

Подробная документация API в `docs/API.md`.

Техническое задание для фронтенда в `docs/TECH_TASK.md`.
